# Фінальний проєкт `goit-algo-fp`

Цей репозиторій містить розв'язання фінального проєкту з курсу «Алгоритми». У проєкті реалізовано сім завдань: робота з однозв'язним списком, рекурсивний фрактал, алгоритм Дейкстри, візуалізація бінарної купи, обходи дерева DFS/BFS, жадібний алгоритм і динамічне програмування, а також метод Монте-Карло.

## Встановлення та запуск

Створіть і активуйте віртуальне середовище:

```bash
python -m venv .venv
source .venv/Scripts/activate
```

Для Linux/macOS можна використати:

```bash
source .venv/bin/activate
```

Встановіть залежності:

```bash
python -m pip install -r requirements.txt
```

Кожне завдання запускається окремим файлом:

```bash
python task_1.py
python task_2.py
python task_3.py
python task_4.py
python task_5.py
python task_6.py
python task_7.py
```

Для завдання 2 можна передати рівень рекурсії аргументом:

```bash
python task_2.py 8
```

Якщо аргумент не передано, програма попросить ввести рівень рекурсії вручну.

## Завдання 1. Структури даних. Однозв'язний список

У файлі `task_1.py` реалізовано однозв'язний список та функції:

- `reverse_linked_list(head)` — реверсує список, змінюючи посилання між вузлами;
- `merge_sort(head)` — сортує однозв'язний список алгоритмом сортування злиттям;
- `merge_sorted_lists(first, second)` — об'єднує два відсортовані списки в один відсортований список.

### Складність

- Реверсування списку: часова складність `O(n)`, просторова складність `O(1)`.
- Сортування злиттям: часова складність `O(n log n)`, просторова складність `O(log n)` через рекурсивні виклики.
- Злиття двох відсортованих списків: часова складність `O(n + m)`, просторова складність `O(1)`, оскільки змінюються посилання між уже наявними вузлами.

## Завдання 2. Рекурсія. Фрактал «дерево Піфагора»

У файлі `task_2.py` створено програму, яка за допомогою рекурсії візуалізує фрактал «дерево Піфагора». Користувач може вказати рівень рекурсії через аргумент командного рядка або через введення з клавіатури.

### Складність

Для кожного рівня рекурсії створюються дві нові гілки, тому кількість викликів зростає експоненційно. Часова складність приблизно `O(2^n)`, де `n` — рівень рекурсії. Просторова складність рекурсивного стеку — `O(n)`.

---

## Завдання 3. Дерева. Алгоритм Дейкстри

У файлі `task_3.py` реалізовано алгоритм Дейкстри для зваженого графа. Для оптимізації вибору наступної вершини використано бінарну купу через стандартний модуль Python `heapq`.

Функція:

- `dijkstra(graph, start)` — повертає найкоротші відстані від стартової вершини до всіх інших вершин графа.

### Складність

За використання бінарної купи часова складність становить `O((V + E) log V)`, де `V` — кількість вершин, а `E` — кількість ребер. Просторова складність — `O(V)` для збереження відстаней і черги пріоритетів.

---

## Завдання 4. Візуалізація бінарної купи

У файлі `task_4.py` на основі базового коду з умови створено функцію для візуалізації бінарної купи.

Основні функції:

- `heap_to_tree(heap)` — створює дерево з масиву, який представляє бінарну купу;
- `visualize_heap(heap)` — візуалізує купу як бінарне дерево;
- `draw_tree(tree_root)` — малює дерево за допомогою `networkx` і `matplotlib`.

Купа перетворюється на дерево за правилом:

- лівий нащадок елемента з індексом `i`: `2 * i + 1`;
- правий нащадок елемента з індексом `i`: `2 * i + 2`.

### Складність

Побудова дерева з масиву має часову складність `O(n)` і просторову складність `O(n)`.

## Завдання 5. Візуалізація обходу бінарного дерева

У файлі `task_5.py` реалізовано візуалізацію обходів дерева:

- DFS — обхід у глибину;
- BFS — обхід у ширину.

Відповідно до умови, рекурсія для обходів не використовується:

- DFS реалізовано через стек;
- BFS реалізовано через чергу `deque`.

Кожен відвіданий вузол отримує унікальний колір у форматі HEX RGB. Колір змінюється від темного до світлого залежно від порядку обходу.

### Складність

DFS і BFS відвідують кожен вузол один раз, тому часова складність кожного обходу — `O(n)`. Просторова складність — `O(n)` у найгіршому випадку для стеку або черги.

## Завдання 6. Жадібні алгоритми та динамічне програмування

У файлі `task_6.py` реалізовано задачу вибору їжі з найбільшою сумарною калорійністю у межах обмеженого бюджету.

Дані про їжу:

```python
items = {
    "pizza": {"cost": 50, "calories": 300},
    "hamburger": {"cost": 40, "calories": 250},
    "hot-dog": {"cost": 30, "calories": 200},
    "pepsi": {"cost": 10, "calories": 100},
    "cola": {"cost": 15, "calories": 220},
    "potato": {"cost": 25, "calories": 350},
}
```

Реалізовані функції:

- `greedy_algorithm(items_data, budget)` — обирає страви за найбільшим співвідношенням калорій до вартості;
- `dynamic_programming(items_data, budget)` — знаходить оптимальний набір страв для максимізації калорійності.

Для бюджету `100` жадібний алгоритм обирає страви з високим співвідношенням калорій до вартості, але не гарантує глобально оптимального результату. Алгоритм динамічного програмування розглядає всі можливі комбінації в межах бюджету та знаходить оптимальний набір.

### Складність

- Жадібний алгоритм: часова складність `O(n log n)` через сортування, просторова складність `O(n)`.
- Динамічне програмування: часова складність `O(n * B)`, просторова складність `O(n * B)`, де `B` — бюджет.

### Висновок

Жадібний алгоритм працює швидко і простий у реалізації, але може не знайти найкраще рішення, бо орієнтується лише на локально найвигідніше співвідношення калорій до вартості. Динамічне програмування повільніше і потребує більше пам'яті, але гарантує оптимальний набір страв для заданого бюджету.

## Завдання 7. Метод Монте-Карло

У файлі `task_7.py` реалізовано симуляцію великої кількості кидків двох ігрових кубиків. Програма:

1. кидає два кубики багато разів;
2. рахує суму значень на кубиках;
3. підраховує кількість появ кожної суми від `2` до `12`;
4. обчислює ймовірність кожної суми;
5. виводить таблицю порівняння результатів методу Монте-Карло з аналітичними розрахунками;
6. будує графік порівняння ймовірностей.

Аналітичні ймовірності:

| Сума | Ймовірність |
|---:|---:|
| 2 | 2.78% |
| 3 | 5.56% |
| 4 | 8.33% |
| 5 | 11.11% |
| 6 | 13.89% |
| 7 | 16.67% |
| 8 | 13.89% |
| 9 | 11.11% |
| 10 | 8.33% |
| 11 | 5.56% |
| 12 | 2.78% |

### Висновки щодо методу Монте-Карло

Метод Монте-Карло дає наближений результат, тому отримані ймовірності можуть трохи відрізнятися від аналітичних. Це нормально, оскільки симуляція базується на випадкових кидках.

При великій кількості кидків результати Монте-Карло наближаються до аналітичних імовірностей. Найчастіше випадає сума `7`, оскільки для неї існує найбільша кількість комбінацій двох кубиків — `6` з `36`. Найрідше випадають суми `2` і `12`, бо для кожної з них існує лише одна можлива комбінація.

Отже, результати симуляції підтверджують правильність аналітичних розрахунків: розподіл імовірностей має форму, у якій центральні суми трапляються частіше, а крайні — рідше.

### Складність

Якщо кількість кидків дорівнює `R`, то часова складність симуляції — `O(R)`. Просторова складність — `O(1)`, оскільки зберігаються лише лічильники для 11 можливих сум.
